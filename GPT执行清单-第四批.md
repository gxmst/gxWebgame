# GPT 执行清单 · 第四批：成长曲线 + 到顶封神 + 外观商店 + 本地记分板

> 本批做四件事，全部**纯前端、无后端、无新美术资源**（外观全部用像素点在代码里画）：
> 1. 成长曲线随进度递增变难（阶段 1）
> 2. 到最高档"封神"：停止刷比玩家大的鱼（阶段 2）
> 3. 封神后进入永久自由霸主模式，替换限时假封顶（阶段 3）
> 4. 外观商店：皮肤扩充 + 帽子/挂件系统（阶段 4）
> 5. 本地记分板：标题页个人最佳榜（阶段 5）
>
> 玩法主线是一条完整曲线：**越爬越险 → 到最高档封神 → 进入永久自由霸主模式（只怕环境威胁）**。
> 纯前端、Canvas 2D、原生 JS、ES Modules，保持零依赖。
> **请按阶段顺序做，每阶段自检通过再进下一阶段。** 阶段 1~3 是玩法，阶段 4~5 是内容/UI，互不干扰。

## 背景（当前实现的问题，先读懂再改）

1. **难度全程恒定**：`director.js` 里 `RELATION_WEIGHTS` 用固定权重刷鱼，其中 `predator`（比玩家大 1.28~1.82 倍的威胁鱼）恒定约 18%，**不随玩家成长变化**。导致前期后期难度几乎一样。
2. **"霸主考验（APEX）"是个限时假封顶**：到 T6 触发 `beginApex()`，只是限时 30 秒把威胁鱼的 `relation` 临时降级为中立（不追不咬），**但生成器仍在照刷比玩家大的鱼**，30 秒后就 `showVictory()` 进结算。所以玩家体验不到"真正封神、只剩场上都是我能吃的鱼"。
3. 已有 `STATE.ENDLESS`（无尽）状态和 `continueEndless()`，但它是通关后玩家手动点"继续称霸"才进入的，且威胁仍在。

## 分阶段执行（请按顺序做，每阶段自检通过再进下一阶段）

### 阶段 1：成长曲线随进度递增变难（前中期）

**目标**：玩家越大，周围威胁越多越凶、可白吃的饵越少。爬得越高越险。

**做法**（数值全部进 `config.js`，不要写死在逻辑里）：

1. 把 `director` 里固定的 `RELATION_WEIGHTS`（prey/fringe/neutral/predator）改成**随玩家进度动态计算**的权重函数。进度建议用玩家当前档位 `player.tier`（1~6）或 `player.mass` 相对最高档阈值的比例。
2. 随进度上升：
   - `predator`（威胁鱼）比例**逐步升高**（例如从 T1 的 ~10% 提升到 T5 的 ~30%）。
   - `prey` + `fringe`（可吃的鱼）比例**逐步下降**，但**永远保留一个下限**（别让玩家完全没东西吃，否则挫败）。
   - `neutral` 大致维持。
3. 威胁鱼的"凶度"也随进度小幅上升（可调项，进 config）：
   - 最大同时追击者数量 `maxChasers`（现在按 tier 是 1/2/3）可以保留或略增。
   - 威胁鱼体型上限：现在 `makeFish` 里 predator 的 ratio 上限随 tier 从 1.58 提到 1.82，可保留此逻辑或让它更平滑地随进度提升。
4. **饵鱼群**（baitSchool）随进度**降低出现频率 / 减小规模**——前期多、给玩家滚雪球的爽感；后期少，逼玩家主动觅食、承担风险。

**所有新增可调参数集中放进 `config.js` 一个新段**，例如 `difficulty:`，写清注释，方便手动调平衡。给出建议初始值，但要让每个值都能独立调。

**阶段 1 自检**：用固定种子（`?seed=1`）跑，在调试面板确认：玩家从 T1 到 T5 过程中，`predator` 比例确实单调上升、`prey` 比例下降但不为 0。截图或打印各档位的关系比例。

### 阶段 2：到最高档"封神"——停止刷比玩家大的鱼

**目标**：玩家成长到最高档（T6）后，生成器**不再刷任何比玩家大的鱼**，玩家成为真正的顶端掠食者。场上只剩下：① 玩家能吃的鱼；② 环境威胁（渔网、水雷、水母、垃圾等）。

**做法**：

1. 在 `director` 生成逻辑里加一个判断：当玩家已达最高档（`player.tier >= 6` 或达到封神条件）时，**`predator` 关系的鱼一律不再生成**（权重清零），已存在的比玩家大的鱼可以让它们自然游走/被 reap 掉，或降级为中立不主动攻击。
2. 确认此状态下 `resolveCollisions` 里"鱼吃玩家"的判定实际不会触发（因为场上不再有能吃玩家的鱼）。渔网、水雷、水母、垃圾等**环境威胁照常生效**——玩家封神但不是无敌。

**阶段 2 自检**：用调试快捷键把玩家调到 T6，跑一段时间，确认场上不再出现红色威胁标记的鱼，但渔网/水雷仍会出现并能威胁玩家。

---

### 阶段 3：封神后进入"永久自由霸主模式"（替换掉限时 APEX 假封顶）

**目标**：玩家到达 T6 封神后，**不再是限时 30 秒然后结算**，而是**直接、自动进入永久自由巡游模式**：玩家是霸主，一直玩下去，分数持续累积，只有环境威胁（渔网、水雷等，且随时间越来越密）能终结这一局。玩家死亡或主动退出才结算。

**做法**：

1. **移除/改造现有的限时 APEX 逻辑**：到 T6 时，不再进入"倒计时 30 秒的考验"，而是**自动进入自由霸主模式**（复用现有的 `STATE.ENDLESS`，或新建一个语义更清晰的状态如 `STATE.APEX`/`STATE.SOVEREIGN`，命名你定，但行为要是"永久、无倒计时"）。
2. 进入该模式时：
   - 给一个明确的**过场提示/toast**："你已成为海洋霸主"之类，让玩家知道自己封神了。
   - HUD 上原来的"霸主考验 30.0 倒计时"要**改成别的**——比如"霸主巡游时长"正计时，或干脆去掉倒计时，换成"当前得分/存活时间"的强调。不要再显示会归零的倒计时。
3. **环境威胁随时间递增**：这是自由模式唯一的挑战来源，必须做好，否则封神后会无聊。
   - 渔网出现频率随巡游时间**逐步加快**（现有 `updateEndless` 里已有 `endlessNetTimer` 随时间缩短的逻辑，确认并强化它）。
   - 可选：水雷/其他环境威胁也随时间增多。
   - 但要**保持"可生存"**——威胁递增要有节奏感，让高手能撑很久、追求高分，而不是封神后马上被网秒杀。相关递增速率进 `config.js`，可调。
4. **分数持续累积**：自由模式里继续吃鱼、连击、得分照常计入本局总分。这一局的最终分数 = 死亡（或主动退出）时的累计分。
5. **结算**：玩家在自由模式中被渔网/水雷终结，或主动退出，才进入结算界面。结算要能体现"成为霸主后巡游了多久、拿了多少分"。原有的胜利/结算界面按需调整文案。

**阶段 3 自检**：完整玩一局到 T6，确认：① 到 T6 自动进入自由模式并有提示；② 没有会归零的倒计时；③ 渔网随时间变密；④ 一直玩到被网住，能正常结算并显示巡游时长和总分。

### 阶段 4：外观商店（皮肤扩充 + 帽子/挂件系统）

**目标**：把现有的皮肤功能升级成一个真正的"外观养成"系统。玩家用珍珠购买、装备外观。**所有外观都用像素点在 `sprites.js` 里代码绘制，不引入任何图片资源。**

**做法**：

1. **皮肤扩充**：现有皮肤（reef/coral/midnight/koi）继续保留，再新增几套（自定，建议共 6~8 套），每套是一组不同的**配色 + 花纹**方案，全部程序化像素绘制。价格用珍珠（`save.wallet.pearls`），沿用现有解锁存档结构。
2. **新增"配件"层（帽子/挂件）**：这是新机制。在鱼身上**再叠画一层装饰物**（如小帽子、王冠、气泡领结、小挂饰等，3~6 种，像素绘制）。要点：
   - 配件要**跟着鱼的位置、朝向、摆尾一起动**（读现有 `angle` / `bodyTwist`），不能是贴在屏幕上不动的死图。
   - 配件与皮肤**相互独立**：玩家可以任意组合"皮肤 A + 帽子 B"。存档要分别记录"已解锁皮肤""已解锁配件""当前装备的皮肤""当前装备的配件"。
   - 配件同样用珍珠购买。
3. **商店 UI**：现有已有一个"成长商店"（金币升级速度/体力/嘴）。外观商店可以**并入现有商店界面新开一个分页/标签**，或新建一个"外观商店"入口，二选一（你定，保持与现有 UI 风格一致）。要能：浏览所有皮肤和配件、看到已解锁/未解锁与价格、购买、装备/切换。装备后立即在标题页的鱼和游戏里生效。
4. **微弱属性加成（重要，务必克制）**：帽子/挂件带**极小**的永久属性加成（例如某帽子 +2% 速度、某挂件 +3% 体力上限之类），**装备即生效、不消耗**。硬性要求：
   - 加成幅度必须是**个位数百分比**级别，且**集中放进 `config.js` 一个 `cosmeticBonus` 段**，带注释、可调。
   - **绝不能让"没买外观的玩家明显更难"**——加成只是给强迫症玩家一点购买动机，不是通关必需。一个只用默认皮肤的玩家必须能正常玩、正常封神。
   - 加成要接入现有的 `upgradeEffects` 那套机制（和商店升级叠加计算），不要另起一套散乱逻辑。
5. **存档兼容**：老存档（没有配件字段）读进来要有默认值兜底，不能因为新增字段导致老玩家存档报错或丢失。

**阶段 4 自检**：购买一套皮肤 + 一个帽子并装备，确认：① 标题页和游戏里的鱼都正确显示皮肤+帽子；② 帽子跟着鱼游动摆动；③ 珍珠正确扣除、解锁状态正确持久化（刷新页面后仍在）；④ 属性加成幅度确实很小、在 config 里可调；⑤ 清空存档后一切回到默认且不报错。

### 阶段 5：本地记分板（标题页个人最佳榜）

**目标**：在标题页展示"本设备历史最佳成绩榜"（Top N，建议 Top 10）。**纯 localStorage，无后端。** 这是为将来合集入口的在线统一排行榜打基础，但本批只做本地版。

**做法**：

1. 每局结束结算时，把这局成绩（**得分、达到档位、存活/巡游时长、日期**）写入一个本地榜单数组，排序后只保留前 N 条，存进现有 `save` 结构（新增一个 `save.leaderboard` 字段之类）。
2. 标题页展示这个榜单：名次、分数、档位、时长、日期。UI 风格与现有标题页一致。榜单为空时（首次玩）显示友好占位（如"还没有记录，去创造第一条吧"）。
3. **只记录有意义的局**（例如得分 > 0 或存活超过几秒），避免开局立刻死也塞一条 0 分进榜。
4. 与现有的"最高纪录 / 最快通关"展示不冲突——可以整合，也可以是标题页新的一块区域。避免重复展示同一数据造成困惑。
5. **数据结构要为将来的在线榜预留**：字段命名清晰（score / tier / durationMs / date），将来做合集在线榜时能直接复用这套数据往服务器提交。清空进度时榜单一并清除。

**阶段 5 自检**：玩几局不同分数，确认榜单按分数正确排序、只留前 N 条、刷新页面后仍在、清空进度后清空。

## 全局约束 & 最终自检

- **数值可调**：本批所有新增平衡参数（难度曲线、威胁递增速率、饵鱼衰减、外观加成 `cosmeticBonus` 等）**必须集中在 `config.js`**，带中文注释，我要能手动反复调平衡。不要把魔数散落在逻辑里。
- **不破坏现有系统**：环形无缝世界、饵鱼群、昼夜、环境物、成长商店、皮肤、存档等已有功能不能被改坏。
- **性能红线**：不得因为难度提升就无限增加鱼的数量——威胁变多应主要靠"调整生成比例"，鱼总数仍受现有上限约束；配件叠加绘制也要轻量（用现有离屏精灵缓存思路，别每帧重画配件）。保证手机 60 帧。
- **外观加成克制**：帽子/挂件加成必须是个位数百分比、集中在 `config.js` 的 `cosmeticBonus`、装备即生效不消耗，绝不能让默认皮肤玩家明显更难。
- **存档兼容**：新增字段（配件、记分板等）读老存档要有默认值兜底，不能让老玩家存档报错或丢失。清空进度时新字段一并清除。
- **不引入后端、不加新美术资源**：本批纯逻辑/数值 + 程序化像素绘制。
- **测试**：为新增的"难度随进度递增""到顶停刷大鱼""自由模式无倒计时""外观加成计算""记分板排序与截断"等核心逻辑补纯逻辑测试，保持现有测试全绿。
