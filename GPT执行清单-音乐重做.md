# GPT 执行清单 · 音乐重做（只改背景音乐，其它一律不动）

> **范围严格限定：只重做背景音乐。** 游戏其它一切（玩法、渔网、比例、流派、PWA、遥测、部署等）已定版，**禁止改动**。
> 纯前端、Web Audio API 程序化生成、零音频文件、零依赖，保持现状。

---

## 问题（先读懂再改）

当前 `js/audio.js` 的背景音乐（`startMusic` 里，约 256~305 行）实现的是 **drone/pad（持续长音垫）**：
- 几个持续不断的振荡器（`voices` / `voiceWaveforms`）叠在一起长鸣；
- 一个低频 LFO 做缓慢"呼吸"起伏；
- 经过低通滤波器（`filterFrequency` / `filterQ`）把高频削掉。

**结果：低沉、绵长、没有节奏、没有旋律、音色发闷——听感像"机器旁边的轰鸣声"。**

**这不是调参数能救的，是实现思路错了。** 用户想要的是 **街机电子音 / 芯片音乐（chiptune / 8-bit arcade）**，是完全相反的方向。

---

## 目标：改成"街机电子音（chiptune）"风格

方向对齐（关键，别再做成 drone）：
- **有节奏、跳动**：要有稳定的节拍（例如每分钟 100~130 拍），像街机游戏那种蹦跳的律动，而不是一根长音。
- **音色明亮、有颗粒感**：以 **方波（square）/ 脉冲波（pulse）** 为主的复古芯片音色（那种"哔哔"声），**不要**用低通滤波器把高频闷掉，**不要**用 sawtooth 低频长鸣。
- **有简单的旋律线**：一段能循环的、轻快的旋律（arpeggio 琶音或简单主题皆可），配一条简单的低音线（bass）打拍子。
- **轻快、不吵、可长期循环**：虽然是街机风，但音量要低、要适合长时间当背景，不能刺耳或喧宾夺主。默认音量偏低。

## 实现要求

1. **重写 `startMusic` 的发声结构**，从"持续 drone + LFO + 低通"改成"**按节拍触发的短音序列**"：
   - 用一个基于 Web Audio 时钟的**步进音序器（step sequencer）**：预排一小段循环（例如 8/16 步），每一步在正确的时间点触发短促的方波音（带快速的 attack/decay 包络，形成"哔"的颗粒感）。
   - 至少两层：**旋律层（明亮方波，跑一段循环旋律/琶音）** + **低音层（低八度方波或三角波，打拍子）**。可选加一层简单的打击感（用极短噪声或高频短音模拟 hi-hat），但保持克制。
   - 旋律用一个音阶内的音符（例如大调或五声音阶），保证"好听不跑调"，循环几个小节后重复。
2. **调度要用 Web Audio 的精确时间**（`AudioContext.currentTime` + 预排未来若干步），**不要**用 `setInterval` 直接播声（会抖动、卡顿）。用"lookahead 调度器"模式：一个定时器周期性地把"接下来一小段时间内该响的音"排进 audio 时钟。
3. **参数集中进 `config.js` 的 `music` 段**（沿用现有段，替换掉 drone 相关的 `voiceWaveforms/voiceDetuneCents/voiceGains/breathFrequency/breathDepth/filterFrequency/filterQ` 等），改成 chiptune 需要的可调项，例如：`bpm`、`baseVolume`、旋律音符序列、低音序列、音色波形、包络时长等。**全部带中文注释、可手动调**，方便用户自己改旋律/速度/音量。
4. **保留现有的对外行为和接口**，不要破坏：
   - `sceneVolume`（不同界面/场景音量缩放）保留。
   - 设置里的**音乐开关、总音量、静音**继续正常生效（`shouldPlayMusic` / 音量总线逻辑不变）。
   - **切后台/页面隐藏时暂停或静音**的现有逻辑保留（不能切到后台还在响）。
   - 淡入淡出（fadeIn / 场景切换过渡）保留，避免音乐突然砰一声开始。
5. **移动端兼容**：仍靠用户首次交互（点"开始"等）解锁音频，别指望页面一加载就自动放。

## 约束

- **只改 `js/audio.js` 和 `js/config.js` 的 `music` 段**（如需可加测试）。其它文件一律不动。
- 纯 Web Audio 合成，**不加任何音频文件、不加依赖**。
- 保持 `node tests/run-node.mjs` 全绿；若现有音乐相关测试因结构改变而失效，相应更新它们（但不要削弱其它测试）。
- 性能：音序器的调度开销要小，不能因为高频 `setInterval` 或每帧建大量振荡器导致卡顿或内存泄漏（用完的 oscillator 要 `stop` 并断开）。

## 自检

- 起服务器实际听：确认背景音乐是**有节奏、明亮、跳动的街机电子音**，不再是低沉轰鸣的长音；能顺畅循环、不刺耳、音量适中。
- 确认设置里音乐开关、音量、静音都生效；切到后台音乐停止；切回来恢复。
- 桌面 + 手机横屏都试；控制台无报错；测试全绿。
