# 文字地牢 · 大世界地图改造 设计稿 v0.1

> **目标**：把游戏从"菜单里选第几层刷"升级成"在一张世界地图上探索的 ARPG"。
> 参照**暗黑2 的世界结构**（不是抄剧情，是借它成熟的三层骨架）。
> 世界观主题：**暗黑奇幻**（剑与魔法、恶魔入侵、亡灵地狱）——现有五职业（战士/法师/游侠/死灵/德鲁伊）天生契合，零额外世界观编写成本。
>
> ⚠️ **这是项目至今最大的一次结构改动**，牵动进度系统、界面架构、野外/地牢入口。
> **必须渐进式分批做，不能一次掀桌。** 分批路线见文末第九节，请严格按批次推进，每批可独立验证。
> 交付方式：本设计稿交给 AI 分批实现；每批完成后由人工审查 + 玩测，通过再进下一批。

---

## 一、核心理念（一句话）

现在：`主菜单 → 选第 N 层 → 战斗 → 回菜单`（一条直线）。
改造后：`世界地图 → 点击城镇/野外/地牢 → 各自的玩法 → 回地图`（有世界感的探索）。

**关键**：这不是推倒重来。现有的战斗、背包、装备、技能、转生、商店、野外刷怪系统**全部保留**，只是把它们**重新组织进一个"世界地图导航层"**里。相当于给现有系统套一个"世界的容器"。

---

## 二、世界结构：暗黑2 的三层骨架

整个世界由一张**世界地图**串起三类节点：

```
              【世界地图】（小地图，点击节点进入）
              /            |              \
       🏰 城镇/村庄     🌲 野外区域        🏛️ 地牢/副本
        (安全区)         (刷怪+随机事件)    (线性深入+Boss)
          │                 │                  │
       NPC对话           点进去自动刷怪        逐层深入
       接支线任务         途中触发突发事件      挑战区域Boss
       商店/重铸          刷装备+材料          通关解锁新区域
       补给              （现有野外系统进化）  （现有层数系统进化）
```

### 2.1 城镇/村庄（安全区）
- **不刷怪**，是补给和社交中心。
- 承载：**NPC 对话、支线任务板、商店、装备重铸、角色管理**。
- ⭐ 落地要点：把**现有的商店、重铸、角色管理界面，重新组织进"城镇"场景**——不是新做，是搬家 + 换外壳。这让这些功能有了合理的世界位置（"在城镇铁匠铺重铸"比"在菜单里重铸"更有代入感）。

### 2.2 野外区域（刷怪 + 随机事件）
- **现有"野外漫步"系统的进化**：现在是"点一下开始野外刷怪"，改造后是"世界地图上有多个野外点，点击某个野外点 → 进入该区域刷怪"。
- 不同野外区域有**不同的怪物池、掉落倾向、等级范围**（详见第三节分区主题）。
- 刷怪过程中**按概率触发"突发事件"**（见第五节事件卡系统）。

### 2.3 地牢/副本（深入 + Boss）
- **现有"层数挑战"系统的进化**：现在的 1~100 层，变成"世界地图上的副本入口"。
- 每个大区域有自己的副本，深入若干层后是**区域 Boss**，击败解锁下一个大区域。
- 副本是"攻坚线"（线性、有尽头、有 Boss），野外是"刷素材线"（可反复、随机）——两者分工明确，互不取代。

## 三、分区主题（广度的实质来源）

**广度不来自"地图大"，来自"不同区域有不同内容"。** 这是整个改造最重要的设计——它让"刷"有了方向性（"我想要某件装备/某种词条，就去刷对应区域"）。

第一版建议做 **3~4 个大区域**，难度递增，主题鲜明：

| 区域 | 主题 | 世界等级范围 | 怪物特色 | 掉落倾向 |
|------|------|------------|---------|---------|
| 🌲 **腐化林地** | 森林/野兽 | 1~20 | 野兽、狼群、蛛类 | 敏捷/暴击装、物理系词条 |
| 🏜️ **枯骨荒漠** | 沙漠/亡灵 | 20~45 | 骷髅、亡灵、沙虫 | 力量/生命装、防御系词条 |
| 🔥 **炼狱深渊** | 地狱/恶魔 | 45~75 | 恶魔、火元素 | 智力/法术装、火抗、高级词条 |
| ⚰️ **虚空秘境** | 终局/混合 | 75+ | 各系精英、混合 | 传说词条、最高级装备 |

- 每个区域包含：**1 个城镇 + 若干野外点 + 1 个副本（末尾区域Boss）**。
- **区域解锁**：打通前一区域的副本 Boss，解锁下一区域。（这替代了原来"通关第N层解锁第N+1层"的线性解锁。）
- ⭐ 落地要点：区域的"怪物池/掉落倾向/等级"其实就是**现有 config 里怪物模板、掉落权重、楼层数值的重新分组**。不是新造一套战斗，是给现有内容"贴区域标签 + 分组"。

## 四、世界等级（统一进度标尺）

用户构想："地牢等级相当于世界等级"——采纳，并明确定义：

- **世界等级 = 玩家当前解锁内容的进度标尺**，衡量：解锁了哪些区域、怪物多强、掉落多好。
- 它和**角色等级**是两回事：角色等级是单个角色的成长（打怪升级、转生重置）；世界等级是**账号/角色对世界的探索进度**（解锁了哪些区）。
- **转生**：转生提升"世界等级上限 / 区域解锁上限"（延续现有"转生解锁更多层"的逻辑，只是从"层"变成"区域/世界等级"）。
- **野外/副本强度**：绑定当前世界等级——你推进到哪，对应区域的怪就是那个强度（延续现有野外"绑定已解锁层数"的逻辑）。

⭐ 落地要点：现有的 `highestUnlockedFloor`、`floorCap`、转生解锁层数这套逻辑，**重新映射成"世界等级 / 区域解锁"**。底层进度数字可以保留，只是换个语义和展示。

## 五、突发事件系统（"事件卡"框架，不硬编剧情）

**不写死的线性剧情，用"事件卡"数据表**——这是杀戮尖塔、暗黑随机事件的通用做法。让"剧情/事件"变成**可增量填充的数据**，而不是必须一次写完的小说。

### 5.1 事件卡结构（纯数据，加一张卡 = 加一条数据）
```js
{
  id: "wounded_traveler",
  title: "受伤的旅人",
  text: "路边一名旅人倒在血泊中，气息微弱……",
  region: "forest",            // 限定区域，或 "any"
  weight: 10,                  // 抽取权重
  options: [
    { label: "救助他（消耗一次治疗）", outcome: { type: "reward", gold: 50, karma: 1 } },
    { label: "搜刮他的行囊", outcome: { type: "loot", rarity: "uncommon", karma: -1 } },
    { label: "无视，继续前行", outcome: { type: "none" } },
  ],
}
```

### 5.2 事件类型（outcome.type）
- `reward`：给金币/经验/材料
- `loot`：掉一件指定稀有度的装备
- `battle`：触发一场（可能是精英）战斗
- `heal` / `damage`：恢复或损失生命
- `none`：无事发生

### 5.3 触发时机
- 野外刷怪过程中，**每隔若干场战斗按概率抽一张事件卡**弹出，玩家做选择。
- ⭐ **第一版只需放 5~8 张简单事件卡**跑通框架。之后有灵感随时往数据表加，零代码成本。

## 六、支线任务系统（城镇任务板，模板化）

同样**模板化、数据驱动**，不硬编：

```js
{
  id: "cull_the_pack",
  giver: "村长",
  region: "forest",
  title: "清剿狼群",
  description: "腐化林地的狼群袭击了村庄，去消灭它们。",
  objective: { type: "kill", enemy: "wolf", count: 10 },   // 或 collect 某材料
  reward: { gold: 200, experience: 500, item: null },
  repeatable: false,
}
```
- 任务类型：`kill`（杀 N 只某怪）、`collect`（收集 N 个某材料）、`reach`（推进到某区域/层）。
- 在城镇 NPC 处接取，完成后回城领奖。
- ⭐ **第一版只做 2~3 个任务模板**，验证"接任务→打怪→回城交任务→领奖"闭环即可。

## 七、界面设计（在现有界面外套一层"世界导航"）

**不是重做所有界面，是新增"世界导航层"，把现有界面作为"节点内容"挂进去。**

### 7.1 新增界面
1. **世界地图界面**（核心新增）：
   - 一张**小地图**（可以是手绘风的节点图，或简单的区域块 + 图标节点）。
   - 标注各节点：🏰城镇、🌲野外点、🏛️副本入口。
   - 点击节点 → 进入对应场景。未解锁区域灰显。
   - 显示当前世界等级、所在位置。
2. **城镇界面**（新增，但内容是搬家）：
   - NPC 列表（对话/任务）、进入商店、进入重铸铺、角色管理入口。
   - 视觉上是个村庄场景，功能上是现有功能的重新入口。

### 7.2 保留基本不变的界面
- **战斗界面**、**背包/装备界面**（含刚做的装备推荐/一键最强/批量出售）、**角色/技能界面**——逻辑不动，只是改为"从地图节点进入"而非"从主菜单进入"。

### 7.3 导航流
```
世界地图 ⇄ 城镇（→商店/重铸/任务/角色）
   ⇅
 野外点（→刷怪+事件+战斗）
   ⇅
 副本（→层数深入+Boss战斗）
```
战斗/背包等随时可从各场景唤出，结束回到所在节点。

## 八、与现有系统的对接（务必保留，别重写）

| 现有系统 | 改造后的位置 | 改动程度 |
|---------|------------|---------|
| 战斗内核（纯函数） | **完全不动** | 零 |
| 背包/装备/推荐/一键最强/批量出售 | 从地图节点进入，逻辑不变 | 极小 |
| 技能/加点/转生 | 转生改为提升"世界等级上限" | 小 |
| 商店/重铸 | 搬进"城镇"场景 | 中（搬家+换壳） |
| 野外漫步 | 进化为"地图上的野外点" | 中 |
| 层数挑战 | 进化为"区域副本" | 中 |
| 多角色存档 | 每角色独立记录世界进度 | 小（存档加字段） |
| 怪物/掉落 config | 按区域重新分组、贴标签 | 中（重组，非重写） |

**存档兼容**：新增"世界进度 / 已解锁区域 / 任务状态 / 事件历史"字段，老存档读入时给默认值（默认解锁第一区域、进度按原 highestUnlockedFloor 映射），**绝不丢老角色**。

## 九、⭐ 渐进式分批落地路线（最重要，严格按批次做）

**绝对不要一次做完。** 每批独立可玩、可验证，通过再进下一批。

### 第一批：世界地图骨架 + 城镇（把现有系统挂进去）
- 做出**世界地图界面** + **第一个区域（腐化林地）**：1 个城镇 + 1~2 个野外点 + 1 个副本入口。
- **把现有的商店/重铸/角色管理搬进城镇**；野外点接现有野外刷怪；副本接现有层数挑战。
- **世界等级/区域解锁**的存档结构建立，老档迁移。
- ✅ 验证目标：能在地图上点城镇补给、点野外刷怪、点副本挑战，现有功能全部正常，老存档不丢。**这批不加事件/任务，先把"容器"搭稳。**

### 第二批：多区域 + 区域主题 + 区域Boss解锁
- 加齐 3~4 个区域，各有主题怪物池/掉落倾向/等级范围。
- 打通区域副本 Boss → 解锁下一区域。
- 转生逻辑对接"世界等级上限"。
- ✅ 验证目标：能从林地一路解锁推进到深渊，各区掉落倾向有区分（刷得出方向性）。

### 第三批：突发事件系统
- 事件卡框架 + 5~8 张初始事件卡，野外刷怪时按概率触发。
- ✅ 验证目标：野外刷怪时会弹事件、选择有不同结果。

### 第四批：支线任务系统
- 城镇任务板 + 2~3 个任务模板，接取→完成→领奖闭环。
- ✅ 验证目标：能在城镇接任务、打怪推进、回城领奖。

### 第五批（可选，打磨）

## 十、技术与质量约束（沿用现有项目规范）

- 纯前端、Canvas/DOM、原生 JS、ES Modules、零依赖、`dungeon/` 子目录内。
- **战斗内核保持纯函数**（无 DOM/时间/随机），可复现、有单元测试。
- 新系统（世界进度、事件、任务）的核心逻辑同样**写成纯函数 + 补单元测试**。
- 所有数值/区域配置/事件卡/任务**集中进 config**，带中文注释，可手动调。
- **不引入后端**（世界进度仍存 localStorage，为将来统一后端预留字段命名）。
- **不污染合集其他部分**（bigfish、大厅）。
- 每批完成跑全部测试保持全绿，桌面 + 手机横屏验证，控制台无报错。
