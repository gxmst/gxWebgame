# 世界地图 · SVG 沉浸版设计稿（激进版）

> 面向 Grok 的执行设计。目标：把第一批做出来的"节点列表式"世界地图，升级成
> **一张有沉浸感的 SVG 手绘风世界地图**——节点是图标、之间用路径线连接、雾隐未探索区、
> 走过的路点亮。**纯表现层改造：只动渲染与交互，绝不改世界数据结构、战斗内核、存档。**
>
> 纯前端、Canvas/SVG、原生 JS、ES Modules，零依赖。

---

## 〇、这一版只做什么 / 绝不做什么

**只做（表现层 + 交互层）：**
- 把世界地图从"按钮/列表"渲染成 **SVG 地图**：区域为块面、节点为图标、节点间有连线路径。
- 视觉沉浸：羊皮纸/暗黑手绘底、迷雾遮罩未解锁区、当前位置高亮、路径流动感、悬停信息卡。
- 保留**列表视图作为可切换的降级视图**（无障碍 / 窄屏 / 出问题时兜底）。

**绝不做（守住边界）：**
- 不改 `world.js` 的数据结构和纯函数（区域/节点/解锁逻辑已定型，只消费它）。
- 不改战斗内核 `combat.js`、不改存档 `save.js` 的结构。
- 不引入任何第三方库（不用 D3、不用地图引擎）——SVG 手写。
- 不做第三批以后的事件/任务系统——本版只是"地图更好看、更好逛"。

---

## 一、为什么用 SVG（而不是 Canvas）

- 地图是**离散节点 + 连线 + 少量区域块**，天然是矢量图形，SVG 直接用 `<path>`/`<circle>`/`<image>` 描述，**可点击、可加 CSS 动画、可无障碍**（每个节点是真实 DOM 元素，能 focus、能读屏）。
- 不需要每帧重绘（地图是静态的，只在移动/解锁时更新），**性能开销几乎为零**，手机无压力。
- 想要"流动的路径线、脉动的当前节点、迷雾渐隐"这些沉浸效果，SVG + CSS 动画就能做，不用写渲染循环。

> Canvas 更适合大鱼吃小鱼那种每帧动的实时画面；世界地图这种"静态可交互图"，SVG 是更优解。

## 二、视觉设计（激进版：暗黑羊皮纸手绘风）

**整体基调**：一张被战火侵蚀的古旧地图，暗黑奇幻。参考暗黑2/杀戮尖塔的地图观感。

- **底图**：深褐/墨绿的羊皮纸质感（用 SVG 渐变 + 噪声滤镜 `feTurbulence` 做旧纸纹理，零图片资源）。边缘做暗角/焦痕。
- **区域块**：每个区域是一块不规则的地形色块（林地=墨绿、荒漠=土黄、深渊=暗红、虚空=紫黑），用 `<path>` 勾出边界，未解锁区盖一层**迷雾**（半透明灰 + `feGaussianBlur`），只露出模糊轮廓和一个"？"，营造"未探索"的神秘感。
- **节点图标**（emoji 或 SVG 图形，跟游戏一致）：
  - 🏰 城镇/村庄（安全区，暖色光晕）
  - ⚔️ 野外刷怪点（可反复进）
  - 🏛️/💀 副本入口（深色，Boss 副本更醒目）
  - 当前所在节点：**脉动光环 + 一个小人物/旗标**标记。
- **连线路径**：节点之间用**虚线或手绘感的曲线路径**连接（`<path>` + `stroke-dasharray`）。已走过的路径**点亮为实线金色**，未走的暗淡。可选：路径上做**缓慢流动的光点**（`stroke-dashoffset` 动画），让地图"活"起来。
- **迷雾解锁动画**：解锁新区域时，迷雾**渐隐消散**（opacity 过渡），有"揭开地图"的仪式感。

**沉浸的关键细节（激进项，酌情做）：**
- 悬停/点击节点：弹出一张**羊皮纸风信息卡**（节点名、类型、推荐世界等级、"进入"按钮）。
- 地图可**轻微拖动/缩放**（pan/zoom），大地图能逛（第一批区域少可先不做，留接口）。
- 环境点缀：地图上散布静态装饰（枯树、骸骨、废墟 emoji/图形），不可交互，纯氛围。
- 当前区域轻微**呼吸式高亮**，锁定区**去饱和 + 压暗**。

## 三、交互与信息架构

**进入世界地图**（从主菜单/城镇"查看地图"）→ 看到 SVG 大地图：
- 已解锁区域清晰、可点；锁定区雾隐 + 显示解锁条件（`unlockHint`）。
- 点击**已解锁区域** → 该区域展开/聚焦，显示区内的城镇/野外/副本节点。
- 点击**节点**：
  - 🏰 城镇 → 进入城镇界面（商店/重铸/NPC，复用现有）。
  - ⚔️ 野外 → 进入野外刷怪（复用现有 outdoor）。
  - 🏛️ 副本 → 进入层数挑战（复用现有 dungeon 战斗）。
- 当前所在节点/区域**始终高亮**，玩家一眼知道"我在哪"。

**视图切换（重要，务必做）：**
- 地图右上角一个切换按钮：**🗺️ 地图视图 ⇄ ☰ 列表视图**。
- 列表视图 = 第一批已有的那套（节点按钮列表），作为**降级/无障碍/窄屏兜底**。
- 用户选择记进 `settings`（localStorage），下次保持。
- **手机窄屏默认可用地图，但列表随时能切**——SVG 地图在小屏要能缩放/滚动查看。

## 四、技术实现要点（给 Grok 的约束）

1. **数据来源不变**：地图渲染只**读** `world.js` 暴露的区域/节点/解锁状态（`listRegions`、节点数据、`unlockedRegions`、`currentRegionId/currentNodeId`）。**不新增也不修改世界数据结构**。若节点需要坐标（x/y）来布局，坐标作为**表现层配置**加在 `config.js` 的地图节点里（或单独的布局表），属于"给已有节点补充显示坐标"，不改逻辑字段。
2. **新建 `js/world-map-view.js`**（或类似）：负责把世界状态渲染成 SVG DOM、绑定节点点击、处理视图切换。**渲染与逻辑分离**——它只调 `game.js` 已有的"进入节点/区域"handler，不自己改状态。
3. **SVG 结构**：一个 `<svg viewBox>` 容器，分层 `<g>`：底图层 / 区域块层 / 迷雾层 / 路径连线层 / 节点层 / 装饰层。层次清晰，便于分别控制。
4. **节点坐标布局**：第一批区域节点少，**坐标可手工配置**（在 config 给每个节点一个 `{mapX, mapY}`，viewBox 归一化坐标）。别做自动布局算法（过度设计）。留好"加区域就加坐标"的扩展位。
5. **纹理/滤镜零资源**：羊皮纸、迷雾、暗角全用 **SVG 原生 `<filter>`（feTurbulence/feGaussianBlur/feColorMatrix）+ 渐变**实现，不引入任何图片文件（保持零资源、秒加载）。
6. **动画克制**：路径流光、当前节点脉动、迷雾渐隐——都用 CSS 动画，**尊重 `prefers-reduced-motion`**（关掉动画仍可用）。别做会让手机掉帧的重特效。
7. **无障碍**：每个节点是可 focus 的按钮（`<a>`/`<button>` 或带 `role`/`tabindex` 的 SVG 元素），有 `aria-label`（节点名+类型+锁定状态）。列表视图是完整的无障碍兜底。
8. **响应式**：SVG `viewBox` 自适应容器；窄屏地图可缩放/横向滚动查看；切换按钮始终可达。

## 五、验收自检（Grok 做完自己跑一遍）

- [ ] 点击城镇/野外/副本节点，**正确进入对应的现有界面**（商店/野外/战斗），行为和列表视图完全一致。
- [ ] 当前所在位置有明显高亮，玩家能一眼定位。
- [ ] 地图⇄列表视图切换正常，选择被记住（刷新后保持）。
- [ ] 桌面 + 手机横屏 + 手机竖屏都能用；窄屏地图可缩放/滚动。
- [ ] `prefers-reduced-motion` 下动画关闭、地图仍完全可用。
- [ ] **未改动**：`world.js` 数据结构、`combat.js`、`save.js` 结构；bigfish 与大厅未受影响。
- [ ] 现有测试全绿；若给地图布局/视图状态加了纯逻辑（如坐标校验、视图偏好 sanitize），补对应测试。

## 六、落地建议（给你，非给 Grok）

- 这一版是**纯表现层**，风险低、最能直接提升你说的"沉浸感"，适合一轮做完。
- 但"激进版"里第二节那些点缀（拖动缩放、流光、环境装饰）是**加分项不是必需项**——建议让 Grok**先把"SVG 节点地图 + 迷雾 + 视图切换"的核心做扎实**，点缀作为"有余力再加"。你玩过核心觉得不够炫，再让它加流光/装饰。
- 交给 Grok 时可以说："先做第一~四节的核心（SVG 地图 + 迷雾 + 视图切换 + 正确进入节点），第二节的拖动缩放/流光/装饰作为可选增强，核心稳了再加。"
