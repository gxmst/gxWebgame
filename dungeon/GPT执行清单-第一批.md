# 文字地牢 · GPT 执行清单 第一批（战斗内核 + 最小可玩循环）

> 先读同目录 `设计稿.md`（v0.2），本清单是它的第 1~2 步落地。
> 位置：合集子目录 `dungeon/`，独立运行、独立存档，**不影响 `bigfish/` 和根目录大厅**。
> 技术栈：纯前端 · 原生 HTML/CSS/JS · ES Modules · **零依赖** · 可直接本地 `python -m http.server` 跑。
> 图标全用 **emoji**。第一版存档用 **localStorage**（结构为将来后端预留，字段命名清晰）。
> **本批目标：把「进地牢 → 群体自动战斗 → 掉装备 → 穿上变强 → 再打」这个核心循环跑通，让人能判断好不好玩。**

---

## 本批范围（只做这些，别贪多）

第一版极简守则（务必遵守，别扩张）：**1 个职业（战士）、3 主属性、4 装备部位、4 稀有度、一小池词条、10~15 层地牢、群体战斗（玩家 1 vs 怪 3~5）。** 多职业、复杂技能树、后端等**本批一律不做**。

### 1. 战斗模拟器内核（最重要，写成纯函数 + 单元测试）
- 写一个**纯函数式战斗模拟器**：输入（玩家状态、敌人群状态、随机种子）→ 输出（完整战斗日志数组 + 结果：胜/负、掉落前的收益数据）。**不碰 DOM、不依赖时间**，纯计算。
- 群体战斗：玩家 1 人 vs 敌人 3~5 只，简单回合制，按速度排序依次出手，一方全灭结束。
- 严格实现设计稿**第六节「群体战斗兜底清单」的全部 7 类边界**（目标选择、出手顺序与 tie-break、回合上限防死循环、群攻平衡、技能冷却、死亡清场时机、空/异常 clamp）。**这是本批质量的关键，逐条落实。**
- 伤害公式按设计稿 4.2，数值进 config。用**可复现的种子 RNG**（同种子同结果，便于测试）。
- 战士技能第一版按设计稿 5.2：普通攻击（0冷却兜底）、重击（单体）、旋风斩（群体）、格挡（增益）。配 5.3 的简单施法 AI（怪多放群攻、怪少放单体、有冷却兜底）。
- **为战斗内核写纯逻辑单元测试**（沿用 bigfish 的 `tests/run-node.mjs` 那套无依赖测试风格）：覆盖兜底清单里的关键边界——回合上限触发、目标死亡重选、群攻打多个、伤害下限、速度 tie-break 可复现等。

### 2. 最小可玩循环（把内核接上极简界面）
- **角色面板**：战士的职业/等级/属性/4 件装备栏/生命值，emoji 图标。
- **地牢选择**：一列层数（第一版可先 3~5 层就够验证循环），标注难度，已解锁才能进。
- **战斗界面**：双方 emoji + 血条 + **滚动战斗日志**（逐行吐出），带 **加速 1×/2×/3× 和 跳过**（设计稿已定）。日志偏长有细节但群攻合并成一条，别刷屏。
- **结算**：胜→给经验/金币 + **掉落一件装备**（本批可先做「随机稀有度 + 基础属性」的简版掉落，词条系统留第三批做全，但结构预留）；败→按方案 B 掉部分经验/金币。
- **装备穿脱**：掉的装备能穿上、能对比「新 vs 旧」属性差异（↑绿↓红），穿上后属性/战力立即变化 → **让「变强」看得见**。
- **存档**：localStorage 存角色、装备、已解锁层、金币；刷新后保留。结构干净，为后端预留。

## 关键要求

- **战斗内核与界面分离**：内核是纯函数（可测、可复现），界面只负责把日志播出来 + 收集玩家的战前操作。别把战斗逻辑写进 DOM 事件里。
- **数值全进 config**：伤害公式系数、技能倍率/冷却、怪物随层成长、掉率、经验曲线、死亡惩罚比例、敌人数量、回合上限等，**集中在一个 config 文件，带中文注释**，方便反复调平衡。
- **严格遵守设计稿第六节兜底清单**——这是群体战斗不出 bug 的保证。
- **极简优先**：本批的目标是验证「循环好不好玩」，不是做全。简版能跑通、能玩、能测，比功能全更重要。

## 自检（做完自己验证）

- `node dungeon/tests/run-node.mjs`（或你建立的测试入口）战斗内核测试全绿。
- 本地起服务器，完整走一遍：进地牢 → 看群体自动战斗（日志能加速/跳过）→ 赢了掉装备 → 穿上变强 → 打更深一层 → 输了掉经验金币 → 刷新页面存档还在。
- 桌面 + 手机浏览器都能正常显示和操作，控制台无报错。
- 确认没影响到 `bigfish/` 和根目录大厅（它们照常能跑）。

## 交付方式

一轮内完成上述实现 + 自检，报告：战斗内核怎么组织的、兜底清单每条怎么处理的、哪些参数可调、循环是否跑通。**先不用管提交推送**（用户会统一处理）。
