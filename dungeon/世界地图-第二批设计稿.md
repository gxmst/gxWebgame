# 世界地图 · 第二批设计稿：让地图「活起来」（事件 + 任务 + 第二区域）

> 前置：第一批（地图骨架 + 城镇挂载）和 SVG 沉浸地图已完成并上线。
> 本批目标：给这张地图**填内容**——野外有意外、城镇有 NPC 和任务、世界能变大。
> 深度定位：**中等**（含多步任务链 + NPC 对话树），战斗**复用现有战斗内核**，不做新战斗机制。
>
> 技术底线（与前几批一致，务必遵守）：
> - 纯前端、Canvas/DOM、原生 JS、ES Modules，零依赖。
> - **战斗内核 `combat.js` 一个字都不许改。** 事件里的战斗一律走现有战斗流程。
> - 事件/任务/对话逻辑集中到**纯数据 + 纯函数**模块，可单测、可复现（种子 RNG）。
> - 不碰 `bigfish/`、不碰大厅。
> - 存档**向后兼容**：老角色没有新字段要有默认值兜底，绝不丢档。
> - 所有内容（事件卡、任务、对话、新区域）尽量做成**数据表**，加内容 = 加数据，不改逻辑。

## 一、本批三大块

1. **野外突发事件（事件卡系统）** —— 野外刷怪时按概率触发随机事件。
2. **城镇 NPC + 任务系统（含任务链 + 对话树）** —— 城镇有 NPC，给任务、讲两句话。
3. **第二区域「荒漠」解锁** —— 打通林地副本 Boss 后开放，世界真正变大。

## 二、野外突发事件（事件卡系统）

### 2.1 触发时机
- 在**野外节点**刷怪的过程中，每清掉一波怪（或每隔 N 秒/N 波），按概率 `eventChance`（进 config，建议 ~20%）触发一张随机事件卡。
- 触发时**暂停自动刷怪**，弹出事件卡面板，等玩家做选择后再继续。
- 城镇、副本内**不触发**野外事件（各有各的节奏）。

### 2.2 事件卡数据结构（纯数据，加卡=加数据）
每张事件卡是一条数据：
```js
{
  id: "treasure_chest",
  title: "锈蚀的宝箱",
  emoji: "📦",
  text: "藤蔓缠绕的宝箱半埋在落叶里，锁扣已经朽坏。",
  // 出现条件（可选）：限定区域、世界等级范围等
  regions: ["forest"],          // 不填=所有区域
  minWorldLevel: 0,
  weight: 10,                   // 随机权重
  options: [
    {
      label: "撬开它",
      // 结果可以是多种效果的组合
      outcomes: [
        { type: "loot", rarityBias: "uncommon" },   // 掉一件装备
        { type: "gold", min: 20, max: 60 },
      ],
      resultText: "箱子里有一些金币和一件装备。",
    },
    {
      label: "小心有诈，绕开",
      outcomes: [],
      resultText: "你绕开了宝箱，什么也没发生。",
    },
  ],
}
```

### 2.3 第一版事件卡（5~8 张起步，覆盖不同"味道"）
- **📦 宝箱**：撬开(得装备/金币，小概率触发精英伏击) / 绕开
- **🧙 神秘商人**：花金币买随机装备 / 离开
- **⚔️ 精英伏击**：无法逃避，**触发一场"精英怪战斗"**（复用现有战斗，怪更强、掉落更好）
- **⛲ 古老祭坛**：献祭金币换永久小 buff（或一次性大额经验） / 无视
- **🩸 受伤的旅人**：救助(耗费一点资源，得感谢/后续任务线索) / 无视 / 趁火打劫(得金币，但……留个道德钩子)
- **🍄 诡异蘑菇**：食用(随机好/坏结果，赌一把) / 不吃
- **💀 无主尸骸**：搜刮(得材料/金币，小概率惊动亡灵触发战斗) / 走开

### 2.4 事件结果的效果类型（outcome types）
统一几种，事件卡组合调用：
- `loot`：掉一件装备（可带 rarityBias）
- `gold` / `experience`：给金币/经验
- `material`：给材料（若材料系统这批做，见第五节；否则先留接口）
- `buff`：永久或临时小增益（幅度要小，进 config）
- `battle`：触发一场战斗（指定敌人配置，复用现有战斗内核）
- `heal` / `damage`：回血/扣血（谨慎，别做成劝退）
- `questFlag`：设置一个任务标记（连接任务系统，见第三节）

### 2.5 事件战斗（精英伏击等）如何复用现有战斗
- 事件里的战斗，**构造一个敌方波次配置，丢给现有的战斗流程**，和野外刷怪、副本战斗走同一套 `combat.js`。
- 差异只在"敌人配置"（精英怪：属性倍率↑、掉落倍率↑），这是数据，不是新机制。
- 战斗结束回到事件结果 → 再回到野外刷怪。

## 三、城镇 NPC + 任务系统（含任务链 + 对话树）

### 3.1 城镇界面
- 进入城镇节点 → 城镇场景：列出该城镇的 **NPC 列表**（图标 + 名字 + 有没有可接/可交任务的标记，比如 ❗可接、❓进行中、✅可交）。
- 城镇里还挂着**已有的功能入口**（商店、重铸、角色管理）——把它们组织成城镇里的"设施"，合理化。
- 点 NPC → 打开**对话面板**。

### 3.2 NPC + 对话树（中等深度）
每个 NPC 是一条数据，含简单对话树：
```js
{
  id: "forest_elder",
  name: "林地长老",
  emoji: "🧝",
  town: "forest_town",
  // 对话树：节点式，每个节点几行文字 + 若干选项，选项可跳转/接任务/结束
  dialogue: {
    root: {
      text: "年轻的冒险者，腐化正在吞噬这片森林……",
      options: [
        { label: "我能帮上什么？", goto: "offer_quest" },
        { label: "告辞", end: true },
      ],
    },
    offer_quest: {
      text: "林中的腐狼越来越多。替我清剿 10 只，如何？",
      options: [
        { label: "接下任务", acceptQuest: "cull_wolves", end: true },
        { label: "再考虑", end: true },
      ],
    },
  },
}
```
- 对话树保持**简单**：几个节点、选项跳转/接任务/结束。不做复杂分支变量（那是 RPG 大工程，本批不碰）。
- 对话是**纯数据**，加 NPC/加对话 = 加数据。

### 3.3 任务系统（含任务链）
任务是数据，支持**多步链式**（做完 A 解锁 B）：
```js
{
  id: "cull_wolves",
  name: "清剿腐狼",
  giver: "forest_elder",
  description: "清剿腐化林地中的 10 只腐狼。",
  // 目标类型：击杀某怪 / 到达某区 / 收集某材料 / 通关某副本
  objective: { type: "kill", target: "corrupt_wolf", count: 10 },
  rewards: [
    { type: "gold", amount: 200 },
    { type: "experience", amount: 500 },
    { type: "loot", rarityBias: "rare" },
  ],
  // 任务链：完成后自动解锁下一个任务（可选）
  nextQuest: "investigate_altar",
}
```
- **目标类型**（第一版做 2~3 种就够）：
  - `kill`：击杀 N 只指定怪（野外刷怪时计数）
  - `collect`：收集 N 个指定材料（若材料系统做）
  - `clear_dungeon`：通关某副本
  - `reach_region`：抵达某区域
- **任务进度**：存进角色存档（`quests` 字段：进行中/已完成/进度计数）。击杀、通关等在对应流程里更新计数。
- **任务链**：完成一个任务，`nextQuest` 自动变为可接（或直接接上），形成"剧情线"。第一版做一条 2~3 步的链走通即可。
- **任务面板**：城镇里或全局有个"任务日志"，看进行中的任务和进度。

### 3.4 交任务
- 目标达成后，回到对应 NPC，对话里出现"✅ 交付任务"选项 → 领奖励 → 若有 `nextQuest` 则解锁下一步。

## 四、第二区域「荒漠」解锁

- **解锁条件**：通关腐化林地副本的区域 Boss（第一批里荒漠是灰显占位，本批点亮）。
- **荒漠内容**（和林地平行的一套）：
  - 城镇（沙漠绿洲镇）+ 至少 1 个 NPC + 1 条任务线
  - 野外节点（沙漠区，怪更强、世界等级更高、掉落更好）
  - 副本（沙漠副本 + 区域 Boss，通关解锁第三区）
  - **掉落倾向差异**（体现"分区主题"）：比如荒漠偏爆发/暴击词条、林地偏生存/物理——让玩家"为了某类装备去刷特定区"。
- 荒漠的怪、掉落、事件卡（可复用通用卡 + 加几张沙漠专属卡）都是数据。
- SVG 地图上：荒漠从"雾遮灰显"变为"点亮可进入"，航路连通。

## 五、材料系统（本批可选，建议先留接口）

- 事件和任务里提到了"材料"。材料的用途（洗词条消耗、合成等）是另一个系统。
- **本批建议**：材料先只做"能掉落、能在背包/城镇看到数量"，用途先接到**已有的洗词条**（洗词条可选消耗材料）或先不做用途。
- **不要**这批就做完整的材料加工/合成系统——那是独立的坑，留后面。config 里给开关。

## 六、界面新增

- **事件卡面板**：弹窗式，标题+emoji+描述+选项按钮，选后显示结果文字+继续。风格与现有对话/结算弹窗一致。
- **城镇场景**：NPC 列表 + 设施入口（商店/重铸/角色）。
- **对话面板**：NPC 头像+名字+对话文字+选项。
- **任务日志**：进行中/已完成任务列表 + 进度。
- 全部复用现有 UI 风格（配色、按钮、弹窗动画），不要另起一套视觉。

---

## 七、数据/存档

- 新增存档字段（都要有默认值兜底、老档迁移不丢）：
  - `quests`：{ active: [...], completed: [...], progress: {...} }
  - `eventFlags`：事件触发过的标记（避免一次性事件重复触发，可选）
  - 材料（若做）：`materials: {...}`
- 事件/任务/对话定义全在 config（或独立数据文件），**逻辑与数据分离**。

---

## 八、必须遵守的约束（重申）

- **`combat.js` 战斗内核零改动**，事件战斗走现有流程。
- 事件/任务/对话的**核心逻辑写成纯函数 + 单测**（触发选择、任务进度更新、对话跳转、奖励结算），种子可复现。
- **存档向后兼容**，老角色照常玩、不丢进度。
- 不污染 `bigfish/` 和大厅。
- 性能：事件/任务是低频交互，不构成性能压力；但事件面板别在刷怪循环里反复重建。
- 保持现有测试全绿，为新系统补测试。
