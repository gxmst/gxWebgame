# 文字地牢 · 第六批(体验打磨)· 已实现

> 本批 7 件小而实的打磨,全部完成并自检。**测试 88/88 全绿。**

1. **存档导出/导入**:顶栏 ⤓/⤒ 按钮。导出为 JSON 文件(文件名带日期);导入走完整的
   sanitizeSave 清洗(兼容 v1/v2/v3 结构),覆盖前二次确认,战斗中禁止导入。
   这是迁后端前的防丢档手段。
2. **结算"再战"按钮**:战斗结算弹窗新增第二个按钮——败/撤退时"再战本层",
   胜利且下一层已解锁时自动变成"挑战第 N 层",一键连刷。野外结算不显示。
3. **商店属性对比**:商店卡片改用与背包一致的对比渲染(vs 身上同部位,↑绿↓红),
   买前一眼看出值不值。
4. **批量出售 + 装备锁定**:
   - 物品新增 `locked` 字段(sanitizeItem 兜底,旧档默认未锁定);
     背包卡片有 🔓/🔒 按钮,锁定后单件出售按钮禁用、批量出售自动跳过。
   - 背包工具条新增"批量出售"行:选稀有度上限(仅普通/优秀及以下/稀有及以下),
     一次确认卖出全部符合条件的未锁定装备。
5. **生涯统计**:角色面板新增小节——胜利/战败/最深纪录/野外波次(当前角色维度,
   数据一直在存档里,现在露出了)。
6. **词条成色**:新增 `getAffixRollQuality()`(纯函数):词条值在"该等级可能区间"里的
   百分位。背包卡片显示成色徽标(≥80% 金色/40-79% 普通/<40% 灰),重铸对比弹窗
   每条词条后也标百分比——重铸决策终于有依据了。
7. **程序化音效**(js/audio.js,WebAudio 合成、零资源文件):命中/暴击/闪避/治疗/
   增益/燃烧/掉落/胜利/战败/升级各一声;同类音效 45ms 最小间隔(3× 速度不炸);
   顶栏 🔊/🔇 开关,状态存进 settings.soundEnabled(存档兼容,默认开);
   AudioContext 懒创建,被浏览器挂起时静默跳过,永不影响游戏流程。

## 存档兼容

- 战斗内核零改动,本批不影响任何战斗数值与可复现性。
