## 阶段 1：角色选择系统（多角色存档）—— 必须最先做

**问题**：现在存档是「单角色」结构，换职业会覆盖掉原角色。加新职业前必须先支持多角色，否则丢档。

**做法**：
- 存档结构从「一个 hero」升级为「**角色列表 + 当前选中角色 id**」。每个角色独立保存自己的：职业、等级、经验、属性、技能点/加点、装备、背包、金币、转生数、层数进度。
- **角色管理界面**（游戏入口处）：
  - 展示已有角色列表（职业 emoji + 名字 + 等级 + 转生数）。
  - **创建新角色**：选职业 + 起名（可默认名），进入游戏。
  - **切换角色**：选中另一个角色继续玩，**不影响其他角色的存档**。
  - **删除角色**：带二次确认，删掉不影响别的角色。
- 角色数量可设上限（进 config，如 6~10 个）。
- **⚠️ 存档迁移（关键，别丢档）**：现有的单角色存档（第二批的 v? 结构）读入时，自动包装成「列表里的第一个角色」，等级/装备/转生/进度**一律不丢**。写迁移测试覆盖。

## 阶段 2：新职业 游侠（Ranger）

- **游侠**：敏捷主属性、**高暴击 / 高闪避 / 多段攻击 / 中等生存**，物理远程。补齐力(战士)/智(法师)/敏(游侠)三系，让构筑维度完整。
- 技能分单体/群体/增益三类（延续框架）：
  - 例：瞄准射击（单体高暴击）、多重箭 / 箭雨（群体）、疾步 / 闪避姿态（增益，提升闪避或速度）。
  - 可体现"多段攻击"特色（一次技能命中多次，每次可暴击）——但注意进设计稿第六节兜底（多段也要走伤害下限、暴击、日志合并）。
- **吃的词条不同**：堆敏捷 / 暴击率 / 暴击伤害 / 攻速，和战士(力量/生命)、法师(智力/魔法伤害)区分开。
- 复用现有 `CONFIG.classes` 职业配置框架 + 共用战斗内核/UI，只是数据不同。角色创建时可选战士/法师/游侠。
- 施法 AI（设计稿 5.3）对游侠生效。补游侠相关单元测试。

## 阶段 3：野外漫步（前台自动刷怪模式）

**定位**：区别于"层数挑战"（线性攻坚、解锁进度），野外是"**在已打通的范围内自由刷素材**"。两条线分工，不互相取代。

**做法**：
- 新增一个「野外/自由漫步」模式入口（和"挑战地牢"并列）。
- 进入后，角色**以 1× 速度持续自动打一波波怪**（复用现有群体战斗内核），打完一波自动接下一波，像轻度挂机。
- **强度绑定进度**：野外怪的强度 = 玩家**已解锁的最高层附近**（可配一个范围/系数），保证"野外刷的是你打得动、又不太亏"的怪，**不能强到取代层数挑战，也不能弱到无意义**。
- **掉落**：每只怪按概率掉「金币 + 装备」（掉率/品质进 config，整体略低于/不同于正规层数掉落，让层数挑战仍有独特价值）。**材料**可作为预留掉落项（先只掉、不做用途），或第一版先不掉，你在 config 里留开关。
- **离开页面即停**：切走标签页 / 关闭页面，野外自动战斗**停止且不结算离线收益**（用户明确要求：离开就不记录）。回来重新开始刷。用页面可见性事件（`visibilitychange`）暂停。
- 可随时手动停止/退出野外，结算已获得的收益。
- 战斗日志在野外模式可**精简**（一波一小结，或只显示掉落播报），避免长时间刷屏；加速/跳过照旧可用。
- **和层数挑战共享**：角色属性、装备、技能、经验一致——野外打怪也给经验（可配比例），刷的装备能穿去打层数。

## 阶段 4：词条前缀命名（装备名带形容词）

**目标**：装备名不再是干巴巴的"铁剑"，而是「**最强词条形容词 + 基础名**」，如"嗜血的巨斧""致命的胸甲""灼热的法杖"。一眼看出装备特色，增加收集乐趣（暗黑Like 经典）。

**做法**：
- 掉落生成时，从该装备的词条里挑**最显著的一条**（数值最高 / 稀有度最高 / 特效词条优先），用它的**形容词前缀**拼到基础名前。
- 建一张「**词条 → 形容词**」对照表（进 config）：如 暴击→"致命的"、吸血→"嗜血的"、火焰→"灼热的"、力量→"狂暴的"、防御→"坚壁的"…… 橙装特效词条给最霸气的前缀。
- 无词条的白装保持基础名（或给个"朴素的"之类）。
- 稀有度仍用颜色区分；前缀 + 颜色 + 词条明细共同构成装备的辨识度。
- 命名是**确定性的**（同装备同名字，基于其词条），不引入额外随机。
