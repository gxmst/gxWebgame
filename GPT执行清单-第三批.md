## 阶段 1:边缘真环形无缝(最优先,底层规则,先做先验)

### 问题现象
世界 `wrap: true`(环形),但玩家游到边缘会"被挡住、需要折返",而鱼却能越过边缘。玩家和鱼行为不一致,且违背环形设计意图。

### 目标
让**玩家也真正无缝穿越世界边界**:从最右侧游出 → 立刻从最左侧出现,上下同理。玩家、鱼、镜头、背景、所有渲染层在跨界处都不能穿帮。

### 要做的事
1. **玩家移动**:确认玩家位置更新后走的是 `wrapEntity(player)`(和鱼一致),不要有任何 `clamp` 把玩家锁在世界框内。检查 `updatePlayer` 里 `wrap` 分支是否真的对玩家生效。
2. **镜头**:镜头跟随玩家跨界时,不能被 `clamp` 到世界边界内(那会造成"撞墙"错觉)。环形世界下镜头应允许自由跟随,worldToScreen 用最近映射(wrap 最短距离)。
3. **⚠️ 接缝渲染(关键,别漏)**:当镜头骑在世界接缝上,视野同时覆盖"世界末端"和"世界开头"。以下渲染层都必须在接缝处正确显示两侧内容,不能出现"半屏空白 / 背景断裂 / 鱼群突然消失":
   - 背景图 / 程序化背景装饰(远景、珊瑚、水草、气泡、微粒)
   - 鱼、玩家、特殊物(水母/水雷/网)
   - 所有特效(粒子/尾迹/环)
   - 做法建议:渲染每个可见物时,除了本体位置,还要检测它在 ±worldWidth / ±worldHeight 的"镜像位置"是否落入视野,若是则一并绘制(wrap 环绕渲染)。
4. **背景图**:第一批接入的固定氛围背景图,若是"固定贴屏"模式则本就无接缝问题;若做了跟随/视差,必须处理跨界。确认现状并保证无缝。

### 阶段 1 自检
- 朝一个方向持续游动 ≥1 整圈,画面全程连续,无黑边、无断裂、无鱼群闪现/消失。
- 骑在接缝上静止,四周鱼群和背景正常显示两侧内容。
- 鱼、水雷、水母跨界表现与玩家一致。
- 测试全绿。

## 阶段 2:鱼种丰富化(外观分层 + 新增鱼种)

### 2A. 现有鱼外观彻底区分(解决"都长一样")
现有鱼种:silver 银鱼 / bluefin 蓝尾 / grouper 石斑 / barracuda 梭鱼 / gold 金鱼。
让 `sprites.js` 给**每一种**画出明显不同的**体型轮廓、配色、花纹、鳍/尾形状**,让玩家一眼能分辨种类和威胁等级:
- 体型比例不同(细长 vs 圆胖 vs 扁平)。
- 配色成体系:猎物系偏冷绿/银,威胁系(梭鱼)偏暗色/尖锐,金鱼醒目金黄。
- 花纹:条纹 / 斑点 / 渐变 / 纯色,各不相同。
- 尾鳍/背鳍形状差异化。
- **性能约束**:精灵必须离屏缓存(每种鱼按体型档预渲染到离屏 canvas 复用),**不要每帧重画像素**。不要动 devicePixelRatio。
- 保持像素风格,`imageSmoothingEnabled = false`。

### 2B. 新增 2-3 种鱼(带独特属性)
在 `config.js` 的 species + `entities.js` 的 SPECIES 里新增,并让 director 会生成它们:
1. **胖河豚(puffer)**:慢吞吞、体型中等偏胖、独行。营养/分数中等。视觉圆胖有刺。(可选:被吃或靠近时鼓起,纯视觉)
2. **深海发光鱼(lantern)**:体型小到中,身上有发光点,**偏好在夜晚出现/夜晚更亮**(和阶段 3 昼夜联动)。分数略高。
3. (可选第三种,你判断)**沙丁鱼(sardine)**:极小、成大群、逃得慢——正好复用/强化饵鱼群机制,作为"一口吞一片"的主力猎物群。
   - 注意:第二批代码里 `game.js:1137` 有个 `fish.species === "sardine"` 的残留分支,请一并理顺:要么正式启用 sardine 作为饵鱼群鱼种,要么清理该死代码。二选一,别留半吊子。

新增鱼种同样要:外观差异化 + 离屏缓存 + 进 director 生成权重(别破坏现有难度平衡,新增鱼以补充多样性为主,占比适中)。

### 阶段 2 自检
- 屏幕上能同时看到多种明显不同长相的鱼,一眼可分辨。
- 新增鱼种能正常生成、被吃、参与碰撞。
- 帧率无明显下降(离屏缓存生效)。
- 测试全绿。

## 阶段 3:昼夜循环 + 玩法联动

### 3A. 昼夜视觉循环(复用现有深度光照系统)
- 加一个全局昼夜时间(周期建议 config 可调,比如一个完整昼夜 3-4 分钟)。
- 整体色调周期循环:**白天(明亮青蓝)→ 黄昏(暖橙)→ 夜晚(深蓝暗)→ 黎明**。
- 复用现有的深度光照 / lighting 配置,叠加一层"时段色调",别推倒重写。
- 夜晚:整体压暗、光束减弱;发光鱼(lantern)、金鱼等的发光点更醒目;可加微弱的鱼群荧光点缀。
- 平滑过渡(lerp),不要突变。所有时段颜色和时长进 config。

### 3B. 玩法联动(你选了"昼夜+玩法联动")
夜晚带来实际玩法差异(保持克制,别把游戏搞太难):
- **视野变窄**:夜晚镜头可见范围/清晰范围略微收缩(比如相机 zoom 略拉近,或远处鱼的关系提示更晚出现),营造"看不太清"的紧张感。
- **夜晚险食/进食得分加成**:夜里吃鱼分数倍率略升(config 可调,比如 ×1.15),奖励夜间冒险。
- **lantern 发光鱼夜晚更多**:夜间生成权重上调,作为夜晚的标志性猎物。
- HUD 或氛围上给个轻提示(如切入夜晚时一句 toast:"夜幕降临")。
- **所有联动强度都进 config**,方便调平衡。

### 阶段 3 自检
- 能观察到完整昼夜色调循环,过渡平滑不突兀。
- 夜晚视野/得分/发光鱼的联动确实生效且可感知,但不至于让游戏变得难玩或烦躁。
- 昼夜不影响帧率。
- 测试全绿(昼夜时间推进、时段色调计算等纯逻辑加测试)。

## 阶段 4:环境物(障碍 / 减速 / 追踪危险 / 收集)

### 4A. 海草障碍(挡路,不致命)
- 在世界里生成成片海草丛(跟随世界坐标、处理跨界)。
- 玩家和鱼穿过时**减速 / 视线受阻**(轻度,别卡死),海草随水流摆动(复用现有摆动/洋流)。
- 密度、减速幅度进 config。

### 4B. 海底垃圾(减速 debuff)
- 零星漂浮的垃圾(如塑料袋、碎片,风格化,别太写实脏乱)。
- 玩家碰到短暂减速(比水母的麻痹轻,是"缠住"的拖慢感),有轻微挣脱反馈。
- 减速幅度/持续时间进 config。

### 4C. 追踪水雷(升级现有 mine)
- 现有 `mine` 是静止的。给它加一个**缓慢朝玩家漂移追踪**的行为(速度远低于玩家,可甩开),灵感来自饥饿鲨的深水雷。
- 追踪速度、感知范围进 config。保留现有触发/爆炸逻辑。
- 只在玩家 tier 达到一定档位后出现(沿用现有 tier≥5 之类的门槛)。

### 4D. 贝壳 / 珍珠(收集物,接入现有货币)
- 海底散布贝壳,玩家碰到可**收集,转化为局外珍珠货币**(接入现有 `save.wallet.pearls` / 结算)。
- 稀有度:普通贝壳少量珍珠,偶尔金色珍珠给较多。
- 收集有轻反馈(闪光 + 飘字 + 轻音效,复用现有 effects)。
- 生成密度、珍珠价值进 config。

### 阶段 4 自检
- 四类环境物都能正常生成、跨界正确、与玩家交互(减速/追踪/收集)符合预期。
- 追踪水雷能甩开,不会作弊式黏死玩家。
- 贝壳珍珠正确累加到存档货币,标题页/商店余额更新。
- 环境物数量有上限,不拖累帧率。
- 测试全绿。

## 给 GPT 的总原则(重要)
- **按阶段顺序做**,前一阶段自检不过不进下一阶段。阶段 1(环形无缝)是后续所有内容正确显示的地基,务必先做扎实。
- **克制**:新增内容以"丰富度和氛围"为目标,不要破坏已调好的核心难度和手感。数值全部外露到 config,让人类后续微调。
- **性能第一**:任何新增渲染都要离屏缓存 / 数量上限 / 可见性裁剪,手机不卡是硬指标。
- 不确定的设计取舍,选保守、可玩、可调的那个,并在代码注释里说明。
