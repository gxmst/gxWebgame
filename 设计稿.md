# 大鱼吃小鱼 · 网页版 设计稿 v0.2

> 目标：一个**重玩法、轻画质**的网页小游戏。纯前端（HTML + Canvas 2D + JS），
> 无后端、无账号系统（进度用 localStorage）。电脑鼠标可玩，手机横屏触屏可玩。
> 部署 = 静态文件托管（Cloudflare Pages / Nginx 均可）。

## 一、一句话定位

你操控一条鱼，在海里穿梭：**吃掉比你小的 → 长大；碰到比你大的 → 被吃掉。**
越长越大，从被追杀的小虾米，翻身成为海里的霸主。核心爽点是**成长翻身**，
核心张力是**风险回报的抉择**（要不要冒险去吃那条跟我差不多大的鱼）。

---

## 二、核心循环（Core Loop）

```
出生（最小体型）
   ↓
在海里游动，寻找猎物
   ↓
吃掉更小的鱼 → 体型增长 + 得分
   ↓
体型跨过阈值 → 进入更高"档位"，能吃的种类变多
   ↓
遇到更大的鱼/威胁 → 躲避（风险管理）
   ↓
（死亡）结算得分 → 存最高纪录 → 用积累的货币解锁东西 → 再来一局
```

这个循环 30 秒就能体验完一轮，符合"即开即玩、一局几分钟"的休闲定位。

## 三、玩法丰富化（重点）

### 3.1 体型档位系统（Size Tiers）
不要用连续体型直接比大小，而是分**档位**，让成长有"节点感"和目标感。
建议 6~8 档，例如：

| 档位 | 名称 | 能吃 | 被谁吃 |
|------|------|------|--------|
| T1 | 小鱼苗 | 浮游生物、比自己小的鱼 | T2+ 几乎所有鱼 |
| T2 | 小鱼 | T1 | T3+ |
| T3 | 中鱼 | T1~T2 | T4+ |
| T4 | 大鱼 | T1~T3 | T5+ |
| T5 | 掠食者 | T1~T4 | T6、鲨鱼 |
| T6 | 霸主/鲨鱼 | 几乎所有 | 仅环境威胁（鱼钩、炸弹） |

规则：**能吃"体型值 < 你 × 系数"的目标**（系数比如 0.9，留一点"擦边不能吃"的紧张感）。
档位只是给玩家的**可视化反馈**（换模型/换颜色/长大一截），底层判定仍是体型数值比较。

### 3.2 风险回报机制（游戏性的核心）
- **擦边鱼**：跟你体型接近、但略小的鱼，吃了成长收益高，但靠近时如果判断失误（对方其实略大）就是送死。制造"要不要赌一把"的抉择。
- **冲刺（Dash / Boost）**：短按触发加速，用来追猎物或逃命。但**消耗体力条**，且冲刺时**转向变笨重**（惯性大），冲过头可能撞进大鱼嘴里。高风险高回报。
- **鱼群诱饵**：一群小鱼聚在一起，吃光收益高，但聚集处往往也是大鱼的猎场。

### 3.3 特殊鱼 / 物件（增加变数，避免单调）
- **金色鱼**：稀有，吃了给大量分数 / 货币。
- **河豚（毒鱼）**：看着能吃，吃了会中毒（短时间减速 / 体型缩水）。教玩家"不是小的都能乱吃"。
- **水母（障碍）**：碰到被蜇，短暂麻痹不能动。不能吃、只能躲。
- **气泡 / 加速带**：环境元素，顺流加速。
- **道具漂浮物**：临时增益（见 3.4）。

### 3.4 临时增益（Power-ups，掉落或吃特殊鱼获得）
- **无敌泡泡**：几秒内不会被吃。
- **磁铁**：一段时间自动吸附附近可吃的小鱼。
- **狂暴**：短时间内能吃比自己大一档的鱼。
- **护盾**：抵挡一次致命攻击（被大鱼咬到不死，但护盾碎）。

### 3.5 环境威胁（后期张力来源）
- **鱼钩 / 渔网**：从上方垂下，被钩到直接死（无视你多大）。让"最强霸主"也不能松懈。
- **炸弹 / 水雷**：碰到爆炸。
- **洋流**：把你往某个方向推，增加操作难度。

### 3.6 进度与解锁（用 localStorage，无需账号）
死亡结算时，把本局得分换算成**货币（珍珠 / 金币）**，累积存本地。可解锁：
- **皮肤**：纯外观，不同颜色/形状的鱼。
- **起始增益（可选）**：比如"出生体型略大""初始体力条更长"——注意别破坏平衡，做成小幅度。
- **新地图 / 主题**：珊瑚礁、深海、沉船，换背景和鱼群构成。

> ⚠️ 设计取舍：解锁项建议**以外观为主、数值增益为辅且幅度要小**，否则老玩家碾压、
> 新鲜感靠数值堆，会掏空"玩法"本身。你想要的是"玩得好才活得久"，不是"氪/肝到大就无敌"。

### 3.7 计分与连击
- 基础分 = 吃掉鱼的体型价值。
- **连击（Combo）**：短时间内连续进食，倍率递增，鼓励激进。断顿则清零。制造"贪吃 vs 保命"的节奏张力。

## 四、关键设计决策（需要你拍板 —— 见文末"待定问题"）

这几个岔路会**从根上影响架构**，必须先定，不然 GPT 写出来方向不对：

### 决策 A：镜头/缩放怎么处理？（最重要）
你的鱼变大后，屏幕怎么表现？三条路：

- **A1（推荐，类 agar.io）**：你的鱼**始终在屏幕中心、大小相对稳定**，靠**镜头拉远 + 周围鱼变小**来体现"你变大了"。世界比一屏大，镜头跟随。
  - 优点：可无限成长、体验最顺、最耐玩。
  - 代价：要实现"世界坐标 ↔ 屏幕坐标"的镜头变换，稍复杂。
- **A2（简单，鱼真的变大）**：世界固定一屏，你的鱼在屏幕上真的越长越大。
  - 优点：实现最简单。
  - 代价：很快占满屏幕，不可持续，只适合"每关有体型上限"的分关卡模式。
- **A3（分关卡）**：每关一个目标体型，达标过关、体型重置。规避无限成长问题。

> ✅ **已定：先 A2 后 A1（两阶段，不是二选一）。**
> 第一版做 **A2 单屏变大**，最快跑通、专心验证"游动 + 吃 + 躲"的手感；
> 手感对了之后，**升级到 A1 镜头跟随**（无限成长、耐玩）。
> 关键：A2 阶段就要**把坐标系写成"世界坐标"**（哪怕世界=一屏），渲染时再转屏幕坐标。
> 这样升 A1 时只改镜头变换那一层，鱼的逻辑一行不动。**别一开始就用屏幕坐标写死，否则升 A1 要重写。**

### 决策 B：世界大小
- 单屏（不滚动）：最简单，鱼群密度高，节奏快。
- 大世界（镜头滚动）：更有"探索海洋"的感觉，配合 A1。

### 决策 C：紧张感机制要不要加"氧气/饥饿"？
- 加**氧气条**（要定期浮到水面换气）或**饥饿**（不吃东西会掉体型）能大幅提升张力，但也提升难度和烦躁感。
- ✅ **已定：第一版不加。** 先把"吃-长-躲"的核心做扎实，觉得不够紧张再加。

## 五、技术难点与实现细节（交给 GPT 时重点标注）

### 难点 1：自适应画布 + 横屏处理
- 画布尺寸**不能写死**，要 `canvas.width = window.innerWidth`（配合 devicePixelRatio 处理高清屏，但注意别过度渲染拖累手机）。
- 监听 `resize` / `orientationchange` 重新计算。
- **竖屏检测**：`window.innerWidth < window.innerHeight` 时，盖一层"🔄 请横屏"提示并暂停游戏；转回横屏自动恢复。
- 网页**无法强制锁横屏**（`screen.orientation.lock` 兼容性差、限制多，不能依赖），只能"检测+提示引导"。

### 难点 2：鼠标/触屏统一控制
- 目标手感：**鱼朝"指针/手指所在位置"游动**（方案 A，最直觉）。
- 统一处理 `mousemove` 和 `touchmove`：抽象出一个"目标点 (targetX, targetY)"，两种输入都只更新这个点。
- 鱼的运动用**朝目标点转向 + 加速度/最大速度限制**，而不是瞬移，才有"游"的手感。加一点**惯性/阻尼**让转向不生硬。
- 触屏注意：`touchmove` 要 `preventDefault()` 阻止页面滚动/缩放；`<meta viewport>` 里禁用用户缩放。

### 难点 3：帧率无关的运动（dt）
- **所有移动必须乘以 dt（帧间隔时间）**，不能直接 `x += speed`。
- 否则高刷手机（120Hz）比普通屏（60Hz）跑得快一倍，游戏速度不一致。
- 用 `requestAnimationFrame` 拿时间戳算 dt，并对 dt 设上限（防止切后台回来 dt 巨大导致穿墙）。

### 难点 4：鱼群 AI（简单但要有层次）
不需要复杂 AI，但要**分行为**才不呆板：
- **小鱼（比你小）**：默认随机游荡（wander）；**你靠近时逃跑（flee）**。逃跑让"捕猎"有追逐感。
- **大鱼（比你大）**：默认游荡；**你进入其感知范围时追你（chase）**。制造威胁。
- **中立**：跟你体型接近的，各走各的。
- 实现：每条鱼一个简单状态（wander / flee / chase），根据与玩家的体型差 + 距离切换。
- （进阶可选）鱼群聚集（boid/flocking），视觉效果好但先别做，属于打磨阶段。

### 难点 5：碰撞检测 + 大小判定
- 判定用**圆形碰撞**（两鱼中心距离 < 半径和）即可，别用精确像素。
- 碰到后比体型：`我的size > 对方size × 系数` → 我吃它；反之它吃我；接近则都不发生（或轻微碰撞）。
- **性能**：鱼多时（几十上百条），两两检测是 O(n²)。第一版鱼不多可以硬来；
  数量上去后用**空间网格（spatial hash grid）** 优化——这正是那个吃鸡游戏 core.js 里的招，
  把世界切格子，只检测邻近格子的鱼。**先别过早优化，等真卡了再上。**

### 难点 6：生成与对象池（Spawn & Pooling）
- 鱼不能无限生成也不能无限存在，要**维持一个合理密度**：屏幕内/世界内保持 N 条鱼，
  死一条补一条，从**镜头外围**生成（别在玩家眼皮底下凭空冒出来）。
- 生成时**按玩家当前体型加权**：既要有能吃的小鱼（正反馈），也要有威胁的大鱼（张力），
  比例随玩家成长动态调整。**这个生成权重表是"手感调校"的核心，会反复调。**
- 高频生成/销毁对象会触发 GC 卡顿 → 用**对象池**（预建一批鱼对象，复用，死了标记 inactive 而非删除）。

### 难点 7：成长与镜头缩放的数值（若选决策 A1）
- 体型增长要有**边际递减**（越大长得越慢），否则前期太慢/后期爆炸。
- 镜头缩放跟随体型：`zoom = 基准 / sqrt(体型)` 之类，平滑过渡（lerp），别突变。
- 世界坐标 → 屏幕坐标的变换要封装干净：`screenX = (worldX - cameraX) * zoom + 屏幕中心`。

### 难点 8：存档（localStorage）
- 存什么：最高分、累计货币、已解锁项、设置（音量等）。**不存"当前对局"**（死了就没了，符合休闲游戏）。
- 读写：`JSON.stringify` / `JSON.parse`，读时给默认值兜底（首次玩没有存档）。
- 边界：用户可能清缓存/无痕模式 → 存档会丢，这是 localStorage 的固有限制，接受它。
- **反作弊无所谓**：纯单机、纯本地，玩家改自己存档不影响别人，不用加密。

### 难点 9：性能（手机是目标机型）
- 目标：中端手机稳定 60 帧。
- 手段：控制鱼的数量上限；避免每帧 `new` 对象；`Canvas 2D` 尽量少状态切换；
  离屏的鱼不画；文字/血条别每帧重建。
- devicePixelRatio 别无脑拉满（高清屏 ×3 会渲染爆炸），封顶 1.5~2。

### 难点 10：手感调校（真正的工作量，非技术）
这些没有"正确答案"，只能反复玩+调，是"游戏性"的真正来源：
- 移动速度、转向阻尼、冲刺力度与冷却。
- 生成权重（能吃的 vs 威胁的比例、稀有鱼概率）。
- 成长曲线、镜头缩放速度。
- 判定宽容度（碰撞半径比视觉模型略小一点，让"我明明躲开了却死了"少发生）。

## 六、开发路线（分步，每步都能在浏览器跑）

1. **地基**：自适应画布 + 一条鱼跟随鼠标/手指游动（含惯性手感）+ 竖屏提示。
2. **鱼群**：随机生成一批大小不同的鱼，各自游荡。
3. **核心循环**：碰撞 + 大小判定 → 吃小的变大、碰大的 Game Over + 计分。
4. **AI 分层**：小鱼见你逃、大鱼追你。生成密度维持 + 对象池。
5. **成长/镜头**：体型档位 + （若选 A1）镜头缩放跟随。
6. **存档**：最高分 + 货币 + localStorage。
7. **丰富化**：特殊鱼、冲刺、增益、连击。
8. **威胁**：鱼钩/水母/炸弹等环境危险。
9. **元系统**：解锁皮肤/主题商店。
10. **打磨**：手感调校、音效（可程序化生成或用少量音频文件）、UI 美化。

> 每一步都是"能玩的东西"，做完一步再叠下一步，避免做到一半卡死。

## 七、技术栈（给 GPT 的约束）

- 纯 **HTML + CSS + 原生 JavaScript**，**Canvas 2D** 渲染。
- **不引入框架**（不用 React/Vue，不用游戏引擎），保持零依赖、单文件或极少文件、可直接双击打开调试。
- 代码结构建议：`index.html` + `game.js`（或按模块拆 `input.js` / `fish.js` / `spawn.js` / `render.js` / `save.js`）。
- 移动用 **dt**，循环用 **requestAnimationFrame**。
- 本地调试：`python -m http.server` 起本地服务器（手机连同 WiFi 用电脑 IP 真机测触屏）。

## 八、美术风格：像素风（已定）

选像素风，理由：纯三角/色块太简陋，手绘卡通又做不好，像素风卡在中间——**低门槛但有辨识度**。

**关键：不用找素材、不用手画，用代码生成。**
- 参照那个吃鸡游戏 `core.js` 的 `makePixelTexture`：在一小块**离屏 canvas**上用 `fillRect` 一格一格拿色块画出鱼，
  再当贴图/精灵用。天然像素风，零美术资源。
- 每种鱼 = 一个"像素图案函数"（几组色块坐标 + 配色）。改配色就能派生不同种类/体型的鱼，复用性高。
- 关键渲染设置：`ctx.imageSmoothingEnabled = false`（关抗锯齿，像素才锐利不糊）；
  逻辑用小尺寸像素网格，绘制时整数倍放大。
- **像素风的好处**：低分辨率 = 每条鱼像素少 = 手机上画得快，跟"性能友好"这条目标天然一致。
- 待打磨阶段可选：像素风的简单帧动画（鱼尾摆动 2~3 帧循环），成本低、活起来效果明显。
